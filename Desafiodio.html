Surgimento do HTML

O HTML (HyperText Markup Language) foi criado no início dos anos 1990 por Tim Berners-Lee, um cientista britânico que trabalhava no CERN (Organização Europeia para a Pesquisa Nuclear), na Suíça.

A ideia surgiu da necessidade de compartilhar documentos científicos entre pesquisadores de diferentes locais do mundo. Para isso, Berners-Lee propôs um sistema de hipertexto que permitisse a conexão entre documentos usando links — daí nasceu a World Wide Web (WWW).

Em 1991, ele publicou a primeira versão do HTML, com apenas 18 tags, dando início à internet como conhecemos hoje.

O que é o HTML?

HTML (sigla para HyperText Markup Language, ou Linguagem de Marcação de Hipertexto) é a linguagem padrão usada para criar páginas da web.

Em vez de ser uma linguagem de programação como Python ou JavaScript, o HTML é uma linguagem de marcação — ou seja, ele estrutura e organiza o conteúdo que aparece no navegador.

Como funciona o HTML?

O HTML funciona como uma “linguagem de marcação”, onde você usa tags para organizar e estruturar o conteúdo da sua página.

Cada tag tem uma função específica, tipo: <h1> para título, <p> para parágrafo, <a> para links.

Essas tags “dizem” ao navegador como mostrar o conteúdo na tela — se é um texto grande, uma lista, uma imagem, etc.

O navegador interpreta o código HTML e renderiza (mostra) a página visualmente para o usuário.

O que são tags?

Tags são os elementos básicos da linguagem HTML. Elas funcionam como “etiquetas” que indicam ao navegador o que o conteúdo representa e como ele deve ser exibido.

Toda tag é escrita entre sinais de menor e maior, tipo: nome-da-tag.

Na maioria das vezes, as tags vêm em pares: uma de abertura <p> e uma de fechamento </p>, envolvendo o conteúdo que você quer marcar.

Algumas tags são auto fecháveis, ou seja, não precisam de uma tag de fechamento, como <img> para imagens ou <br /> para quebras de linha.

Entenda o que cada tag faz e em que situações ela deve ser usada no HTML.

  • <h1> a <h6>: Essas tags definem títulos de diferentes níveis, que são usados para organizar o conteúdo de forma hierárquica.
  • <p>: Define parágrafos de texto, ideal para blocos com uma ou mais ideias.
  • <mark>: Destaca um trecho do texto com um fundo amarelo, como se fosse marca-texto.
  • <small>: Indica texto de menor importância, exibido em fonte reduzida. Muito usado para rodapés, notas ou avisos legais.
  • <i>: Aplica estilo de itálico ao texto, geralmente usado para expressar ênfase leve ou termos estrangeiros.
  • <u>: Sublinha o texto (underline). Usado raramente hoje em dia, mas pode destacar elementos específicos.
  • <strong>: Dá destaque semântico ao texto, indicando que ele é importante (aparece em negrito). Funciona visualmente como a tag <b>, mas com a diferença de que <strong> possui significado semântico, o que ajuda leitores de tela e softwares de acessibilidade a entenderem que aquele trecho merece ênfase.
  • <ol> e <ul>: Criam listas ordenadas (com números) e não ordenadas (com marcadores), respectivamente.
  • <li>: Define um item dentro de qualquer tipo de lista (ol ou ul).
  • <a>: Cria hiperlinks, permitindo a navegação entre páginas ou seções de uma única página.
  • <hr>: Insere uma linha horizontal, usada para separar visualmente conteúdos ou seções.
  • <sub>: Exibe texto como subscrito, útil em fórmulas químicas (ex: H2O).
  • <sup>: Exibe texto como sobrescrito, comum em potências (ex: x2).
  • <blockquote>: Citação em bloco, usada para trechos de textos ou falas com referência (citadas de fontes externas).
  • <img>: Insere imagens na página. É uma tag auto fechável.
  • <div>: Elemento genérico de bloco, usado para agrupar e organizar conteúdo. Muito útil para aplicar estilos com CSS.
  • <br>: Quebra de linha. Usada para pular uma linha sem iniciar um novo parágrafo.
  • <form>: Cria formulários para envio de dados (cadastro, login, etc.).
  • <input>: Campo de entrada de dados dentro de formulários (texto, senha, checkbox, etc.).
  • <table>: Utilizada para criar tabelas com linhas e colunas.
  • <font>: Define cor, tipo e tamanho da fonte. Essa tag já está obsoleta no HTML5, ainda dá para usar, mas com o tempo e com novas atualizações ela será descontinuada.
  • <del>: Mostra que um texto foi excluído. Fica com um risco no meio, indicando remoção.
  • <abbr>: Abreviação com explicação. Ao passar o mouse sobre a sigla, mostra o significado.
  • <section>: Define uma seção genérica do documento, usada para agrupar conteúdo tematicamente relacionado. Ajuda na organização e melhora a navegação assistida.
  • <nav>: Representa uma seção da página com links de navegação, como menus ou índices. Importante para acessibilidade e estrutura semântica.

Conclusão

Até agora, aprendi que o HTML é a base essencial para estruturar páginas web, funcionando como as ferragens em uma construção. Só com HTML é possível criar páginas, mas o resultado costuma ser um conteúdo “bruto”, sem muita formatação ou apelo visual — cumpre o básico, mas não encanta.

É quando o CSS entra em cena que a mágica acontece: ele estiliza e embeleza o site, definindo cores, layouts e fontes, deixando tudo visualmente atraente e organizado. E para tornar a experiência interativa, dinâmica e responsiva, o JavaScript é indispensável, permitindo que os sites “conversem” com os usuários, respondam a ações e criem funcionalidades complexas.

Ainda estou no começo dessa jornada, aprendendo e praticando esses pilares fundamentais do desenvolvimento front-end. Sei que dominar HTML, CSS e JavaScript será o caminho para me tornar um desenvolvedor com conhecimento sólido, capaz de criar sites e aplicações web completas, funcionais e atraentes.

Afinal, não basta ter apenas a estrutura; é preciso estilo e inteligência para construir uma experiência digital de qualidade — do jeitinho que quero oferecer como futuro desenvolvedor.
